# MeetMind 家长端升级规划 v2.0

> 基于 K12 教育产品用户研究洞察（覆盖 5000+ 家长样本）

---

## 核心洞察

### 中国家长陪学的真实痛点

> **中国家长陪学不是"不会题"，而是缺三样东西：**

| 缺失 | 表现 | 产品机会 |
|------|------|----------|
| **确定性** | 不知道孩子到底哪里卡 | 困惑点精准定位 + 老师原话 |
| **效率** | 没时间、想短时间有效陪 | AI 陪学脚本 + 时长估算 |
| **情感满足** | 陪学容易吵架、没成就感 | 情绪化解 + 即时反馈 |

### 现有功能覆盖度

| 断层类型 | 覆盖状态 | 待补齐 |
|----------|----------|--------|
| 信息断层 | ✅ 已覆盖 | 优化老师原话调用体验 |
| 方法断层 | ⚠️ 部分覆盖 | 情绪化解脚本、陪学话术 |
| 反馈断层 | ❌ 未完善 | 趋势报告、即时反馈、成就感 |

### 家长使用时间分布

| 时间段 | 场景 | 核心需求 |
|--------|------|----------|
| 18:00-20:00 | 下班后/晚饭后 | 快速概览（5 分钟了解学情） |
| 20:00-22:00 | 陪学战斗时间 | 即时工具（脚本 + 原话调用） |
| 22:00 后 | 睡前检查 | 进度总结 + 次日提醒 |
| 次日 7:00 | 早间 | 昨日未解决提醒 |

### 不同家长类型画像

| 类型 | 占比 | 特点 | 产品期待 |
|------|------|------|----------|
| 高参与度妈妈 | 60%+ | 情感需求强、内疚感重 | 细腻脚本、情绪引导、进步可视化 |
| 忙碌的爸爸 | 20% | 时间少、结果导向 | 极简概览、数据证明、周末补齐计划 |
| 隔代长辈 | 15%+ | 操作门槛高、视力问题 | 大字体、语音播报、傻瓜式清单 |

## 🔴 高优先级（首发必须 - MVP 后 1-2 周）

### 1. 三张卡模式升级

> 保留原有"三张卡"框架，融入用户研究洞察

#### 断点卡（今天卡住的点）

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 按科目分类 | 数学/英语/语文 Tab 切换 | P0 |
| 老师原话片段 | 每个困惑点显示前后 30 秒讲解，可点击播放 | P0 |
| 困惑严重度标签 | 高频（重复出现）/ 首次 | P1 |
| 一键跳转 AI 解释 | 点击困惑点 → 进入 AI 家教对话 | P0 |

#### 计划卡（今晚怎么陪）

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| AI 陪学脚本 | 具体的陪学步骤和话术 | P0 |
| 预估时长 | 精确到分钟（如"约 25 分钟"） | P0 |
| 情绪化解提示 | 当孩子说"不会"时的温和回应建议 | P0 |
| 分步骤指引 | 第 1 步：一起听原话 → 第 2 步：问哪里卡 → 第 3 步：AI 解释 | P1 |

#### 完成卡（陪学成果）

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 任务打勾 | 可视化勾选每个困惑点 | P0 |
| 进度环动画 | 完成一个点 → 进度环增长 + 庆祝动效 | P0 |
| 即时反馈弹窗 | "太棒了！孩子掌握度提升！" | P1 |
| 家长心情记录（可选） | 😊 顺利 / 😐 一般 / 😤 有冲突 | P2 |

**涉及文件**：
- `src/app/parent/page.tsx` - 主页面重构
- `src/lib/services/parent-service.ts` - 数据逻辑
- `src/components/business/ProgressRing.tsx` - 进度环组件

### 2. AI 陪学脚本优化

> 核心转变：从"讲题"转向"怎么陪"

#### 功能设计

| 功能点 | 说明 |
|--------|------|
| **场景化话术** | 针对每个困惑点，给家长具体的引导话术（而非知识点解释） |
| **情绪化解脚本** | 常见冲突场景 + AI 建议回应方式 |
| **分步骤指引** | 第 1 步：和孩子一起听老师原话<br>第 2 步：问"你觉得哪里卡住了"<br>第 3 步：用 AI 辅助解释 |
| **时长估算** | 每个困惑点预估 X 分钟，总计 X 分钟 |

#### 情绪化解场景示例

| 孩子说 | AI 建议家长回应 |
|--------|-----------------|
| "我不会" | "没关系，我们一起听听老师是怎么讲的" |
| "老师不是这么讲的" | "那我们听听老师原话，看看老师用的什么方法" |
| "我听懂了"（但做不对） | "那我们一起做一道试试，看看哪个步骤卡住了" |
| "这题太难了" | "老师讲的时候也提到这是难点，我们拆开一步步来" |

**涉及文件**：
- `src/lib/services/parent-service.ts` - `generateActionScript()` 方法重构
- 新增情绪化解话术 prompt

## 🚫 警惕伪需求（暂缓或不做）

| 功能 | 为什么是伪需求 | 建议 |
|------|----------------|------|
| **复杂知识图谱** | 家长说"高大上"，实际打开率 <10%，没时间看 | 暂缓，深度功能藏在"查看更多" |
| **AI 心理画像** | 看起来专业，但家长不知道怎么用 | 不做 |
| **强制打卡/日记** | 家长陪学已累，不想额外记录 | 不做，心情记录改为可选 |
| **过多推送** | 超过每天 3 条，家长直接静音 | 严格控制推送频率（≤3 条/天） |
| **纯数据报告** | 长表格/统计图，忙碌家长跳过 | 简化为"一句话总结" |
| **社区讨论** | 家长怕隐私泄露 + 没精力参与 | 初期不做 |

## 📝 附录：用户研究关键发现

### 家长最容易产生获得感的时刻

1. **陪学脚本实际化解冲突** - 按脚本陪，孩子突然懂了 + 不吵架
2. **进度环/任务打勾闭环** - 像游戏通关，有"掌控感"
3. **趋势改善证明** - "上周 5 个困惑，这周减到 2 个"

### 高黏性功能（养成习惯）

1. **智能每日推送** - 固定时间推送"今日陪学预览"，打开率最高
2. **跨日连续反馈** - "连续 3 天解决率 100%，孩子掌握更稳了"
3. **家庭仪式感** - 睡前"亲子总结"（AI 语音播报今天收获）
4. **季节性事件钩子** - 期中考前"冲刺计划"、开学"新学期目标"

### 养成路径建议

```
首周：高频引导（陪跑推送）
  ↓
第 2 周：进度奖励
  ↓
第 1 月：趋势报告
  ↓
目标：30 天留存 > 40%
```
