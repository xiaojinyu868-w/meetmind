## 用户需求

### 需求概述

用户反馈移动端录音界面存在两个 UI 问题需要修复。

### 核心功能

1. **录音状态栏控件放大**

- 当前暂停/停止按钮 56px 太小，需要放大约 1.3 倍至 72px
- 按钮内图标从 28px 放大至 36px
- 时间显示字体从 30px 放大至 36px
- 顶栏高度相应调整以容纳更大的控件

2. **修复移动端 Tab 背景过宽问题**

- 录音/复习切换 Tab 组件的背景容器宽度设置为 180px 固定宽度
- 但 Tab 实际内容只有两个按钮，导致右侧出现明显空白
- 需要让背景自适应内容宽度

## 技术栈

- 前端框架: Next.js + React + TypeScript
- 样式方案: Tailwind CSS
- 现有组件: MobileTabSwitch, Recorder

## 实现方案

### 1. 录音状态栏控件放大

**修改文件**: `src/components/Recorder.tsx`

**尺寸调整方案** (移动端放大 1.3 倍):

| 元素 | 当前尺寸 | 调整后 (移动端) | 桌面端 (sm:) |
| --- | --- | --- | --- |
| 顶栏高度 | h-[72px] | h-[84px] | h-[60px] |
| 暂停/停止按钮 | w-[56px] h-[56px] | w-[72px] h-[72px] | w-[48px] h-[48px] |
| 按钮图标 | w-7 h-7 | w-9 h-9 | w-6 h-6 |
| 时间字体 | text-3xl | text-4xl | text-2xl |
| 状态文字 | text-base | text-lg | text-sm |
| 状态指示点 | w-3 h-3 | w-3.5 h-3.5 | w-2.5 h-2.5 |
| 按钮间距 | gap-3 | gap-4 | gap-2 |


### 2. 修复 Tab 背景过宽

**问题根源**:

- `page.tsx` 第 966 行设置了 `className="w-full max-w-[180px]"`
- 这导致 Tab 组件被强制拉伸到 180px 宽度
- 而 MobileTabSwitch 组件本身使用 `inline-flex`，应该自适应内容宽度

**修改方案**:

- 移除 `w-full max-w-[180px]` 限制
- 让 Tab 组件自然根据内容宽度显示

## 实现细节

### 触摸目标规范

- 修改后移动端按钮为 72px，远超 Apple/Google 推荐的 44px 最小触摸目标
- 确保用户在移动设备上能够轻松点击

### 响应式设计

- 使用 Tailwind 的 `sm:` 前缀区分移动端和桌面端样式
- 移动端 (默认): 更大的触摸目标和字体
- 桌面端 (sm: 640px+): 保持适中尺寸
